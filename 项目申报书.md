# 基于系统级多模态唤起的 AI 智能交互助手研究与实践

---

## 一、项目介绍

### 选题背景

大语言模型（LLM）与智能 Agent 技术的快速普及，正在将 AI 能力从"专用工具"推向"随时可用的基础设施"。然而，当前主流 AI 产品（ChatGPT、Claude、本地 Agent 框架等）普遍存在一个共同摩擦点：**使用它们需要主动切换应用**。用户在撰写文档、编写代码或处理数据时，一旦需要 AI 协助，必须中断当前工作，切换至对应的 AI 应用，输入问题，再将结果手动复制回工作界面。这种"离开工作现场→获取 AI 能力→返回现场"的往返流程，打断了用户的专注状态，显著降低了 AI 辅助的实际价值。

与此同时，语音作为人类最自然的表达方式，在 AI 交互中仍未得到充分利用。现有语音工具大多局限于单纯的语音转文字，缺乏与 LLM 和 Agent 的深度集成，无法将"说话"这一低门槛的输入方式转化为触发复杂 AI 任务的指令入口。

在国家推进"人工智能+"战略和新型人机交互创新的背景下，如何构建一个**无需切换应用、在任意场景下随时唤起 AI 能力**的系统级交互层，是当前 AI 应用融合领域亟待突破的重要课题。

### 项目概述

本项目研发了 **Voke**——一款运行于操作系统层的 AI 智能交互助手。其核心设计理念是：**在任何可以输入文字的地方，均可通过快捷键随时唤起 AI 能力，无需离开当前工作界面**。

系统提供两个层次的功能：

**基础功能——系统级语音转文字**

在操作系统任意输入框（文档、代码编辑器、浏览器、邮件、聊天工具等）中，长按快捷键即可开始语音输入，松开后转录结果直接输出到光标所在位置，无需打开任何额外窗口。相比传统输入法内置的语音功能，Voke 接入云端高精度 ASR 模型（阿里云 DashScope Qwen-ASR），在专业词汇、长句识别和多语言混输方面具备显著优势。

**高级功能——随时随地唤起 AI 与 Agent**

- **一键询问大模型**：选中任意文本或通过语音描述问题，系统调用云端或本地 LLM（兼容 OpenAI 接口的任意服务）即时作答，答案直接注入当前光标位置或悬浮展示，无需切换到 ChatGPT、Claude 等应用；
- **唤起本地 Agent**：通过语音指令调起 Claude Code、OpenClaw 等本地 Agent，将文件整理、数据分析、代码生成等复杂任务直接委托给 Agent 执行，Voke 充当 Agent 的语音调度入口；
- **上下文感知指令**：选中文本后发出语音指令，系统将选中内容与语音命令组合后交给 LLM 处理，实现"对着当前内容说话就能改它"的自然交互体验。

### 选题意义

**交互范式创新**：Voke 将 AI 能力从"需要打开的应用"变为"随时存在的系统能力"，从根本上消除了 AI 辅助的使用摩擦，探索了下一代人机交互的可能形态。

**技术融合价值**：项目深度整合语音感知、LLM 推理、本地 Agent 调度三个技术方向，在跨平台系统集成、多模态指令理解、Prompt 工程等方面形成完整的工程实践，具有较高的技术综合度。

**普惠应用价值**：降低 AI 工具的使用门槛，让非技术用户也能通过自然语音触达复杂 AI 能力，推动 AI 在办公、创作、研究等场景的真正落地，体现 AI 技术的普惠导向。

## 二、项目方案

### 1. 研究内容、方法与实施步骤

#### 研究内容

本项目的研究内容划分为以下四个模块：

**模块一：系统级全局唤起机制**

这是 Voke 区别于普通应用的核心技术。研究如何在操作系统层面实现非侵入式的全局热键监听，使系统能够在用户处于任意应用（浏览器、IDE、Office、聊天工具等）时均可响应唤起指令，而不依赖当前活跃窗口的焦点。采用 `uiohook-napi` 库实现跨平台全局键盘事件捕获，结合 Electron 主进程常驻后台的特性，构建"零感知、随时在线"的 AI 接入层。

**模块二：语音采集与高精度 ASR 转录流水线**

研究基于热键触发的实时音频采集方案：热键按下时启动录音，松开时终止并触发转录。使用 `ffmpeg-static` 对原始音频进行格式转换和降噪预处理，将音频编码为 Base64 后通过兼容 OpenAI 格式的接口提交至 ASR 服务，获取高精度转录文本。转录结果通过剪贴板注入或模拟键盘输入，无缝写入当前光标位置，实现对任意应用的透明兼容。

**模块三：LLM 与本地 Agent 的多模式调度**

设计三套差异化的交互模式及对应的 Prompt 工程方案：

- **润色模式**：将原始语音转录文本交给 LLM 进行口语词去除、语义修复和书面化优化，Prompt 约束模型在优化的同时最大程度保留用户原始表达风格；
- **问答模式**：将用户语音问题（可附带选中文本作为上下文）提交 LLM，答案直接注入当前输入位置，省去打开独立 AI 应用的步骤；
- **Agent 唤起模式**：研究将语音指令解析为结构化任务描述，通过命令行或 IPC 接口调起 Claude Code、OpenClaw 等本地 Agent，实现文件整理、数据分析、代码生成等复杂任务的语音委托。系统兼容任意支持 OpenAI 接口格式的云端或本地大模型（Ollama、LM Studio 等）。

**模块四：数据本地化与隐私安全设计**

所有用户数据（历史记录、配置信息、热词库）均通过 SQLite 存储于本地设备，API Key 等敏感配置在持久化前进行加密处理。系统不设任何自有后端服务器，所有 API 调用直接由客户端发起至用户自选的服务商，从架构上保障数据安全。

#### 研究方法

- **工程实现法**：以实际可运行的系统为交付目标，采用迭代开发模式，快速原型验证核心路径后逐步完善；
- **对比实验法**：对比不同 Prompt 策略下润色效果（流畅度提升、原意保留度、口语词去除率），量化评估 Prompt 工程方案的优劣；
- **用户测试法**：招募不同背景的真实用户，测量使用 Voke 前后完成典型任务（如撰写邮件、查询问题、整理文件）的操作步骤数与时间，量化"减少应用切换"带来的效率提升；
- **文献调研法**：调研系统级 AI 交互、语音 Agent、多模态 Prompt 工程等方向的学术与工程进展，指导关键设计决策。

#### 实施步骤

| 阶段 | 时间 | 主要工作 |
|------|------|----------|
| **第一阶段**：需求分析与技术选型 | 第1-2周 | 调研竞品与同类工具、确定技术栈、搭建 Electron+React+Vite 开发环境 |
| **第二阶段**：全局唤起与语音转文字核心链路 | 第3-5周 | 完成全局热键监听、音频采集、ASR 接入、剪贴板/键盘注入 |
| **第三阶段**：LLM 多模式接入 | 第6-8周 | 实现润色、问答两种模式，完成 Prompt 模板设计与调优，兼容多种 LLM 服务 |
| **第四阶段**：本地 Agent 唤起集成 | 第9-11周 | 研究并实现语音指令→Agent 任务的调度机制，完成 Claude Code / OpenClaw 接入 |
| **第五阶段**：工程完善与测试 | 第12-13周 | 本地数据库、加密存储、悬浮 UI、系统托盘、多平台兼容性测试 |
| **第六阶段**：用户测试与迭代 | 第14-15周 | 招募用户测试、收集效率数据、修复问题、完成打包发布 |
| **第七阶段**：总结与成果输出 | 第16周 | 撰写技术文档、项目报告、准备结项材料 |

### 2. 人员分工

本项目团队由2人组成，根据各成员技术方向进行分工：

| 角色 | 主要职责 |
|------|----------|
| **项目负责人（成员A）** | 整体架构设计与技术决策；Electron 主进程开发（全局热键、窗口管理、IPC 通信）；ASR 音频处理流水线；LLM 调用模块与 Agent 唤起机制；本地 SQLite 数据库设计与实现；项目进度统筹 |
| **成员B** | React 渲染进程与完整 UI 开发（主界面、设置页、历史记录、统计、悬浮覆盖层）；Prompt 模板设计与效果评估实验；用户测试方案设计、数据收集与分析；技术文档与项目报告撰写 |

协作方式：使用 Git 进行版本管理，采用 Feature Branch 工作流，每个功能模块在独立分支开发，经 Code Review 后合并主分支；每周举行进度同步会，及时处理跨模块依赖问题。

## 三、预期成果

### 3.1 软件成果

完成 Voke 的完整开发并发布 Windows 平台安装包（同步适配 macOS 与 Linux），实现以下功能的稳定运行：系统级全局语音转文字、LLM 问答与文本润色、本地 Agent 语音唤起；历史记录、热词管理、使用统计等辅助功能；兼容任意 OpenAI 格式接口的 ASR 与 LLM 服务自由配置。项目代码以 MIT 协议开源于 GitHub，供社区学习与二次开发。

### 3.2 研究成果

量化验证"系统级随时唤起"对 AI 使用效率的提升效果：通过对比实验，统计完成相同任务时 Voke 与传统"切换应用"方式的操作步骤数差异，以及不少于20名用户的主观满意度数据；形成一套面向口语 ASR 转录文本的 Prompt 工程规范，评估不同策略在流畅度与原意保留之间的权衡。

### 3.3 文档成果

撰写完整技术报告，涵盖系统架构设计、全局唤起机制实现方案、Agent 集成接口设计、Prompt 调优记录及用户测试分析，形成可供后续研究复用的工程实践资料。

### 3.4 预期影响

Voke 探索了"AI 能力系统化"这一方向——将大模型与 Agent 从独立应用变为操作系统级的随时可用能力。这一范式若得到推广，将根本性地降低 AI 工具的使用门槛，推动 AI 在办公、创作、编程、数据处理等场景的深度融合，为 AI+ 行业应用提供可复用的系统级集成思路。
